Задача 2. Разработать цифровой аналог автопарка
Условия:
- В автопарке должны быть представлены различные виды транспорта (грузовой,
пассажирский, грузопассажирский).
- У каждого транспортного средства должны быть свои характеристики, как
общие (например, год выпуска, марка и модель, вид используемого топлива, его
расход ...), так и специфичные (например, объём кузова и грузоподъёмность
для транспорта, перевозящего грузы, и пассажировместимость – для
перевозящего пассажиров).
- Аналогично с методами: должны быть как общие (например, заправить и
отремонтировать), так и специфичные (например, продезинфицировать салон
для автобусов и опломбировать кузов для грузовиков).
- Грузовые автомобили должны иметь разные типы кузовов и, соответственно,
разные варианты перевозимых грузов (например, в тентованных можно
перевозить промтовары, в рефрижераторах – промтовары и скоропортящиеся
продукты, в цистернах – жидкости и т.д.).
- Также следует предусмотреть выполнение заказов на перевозки. Заказ должен
включать начальный и конечный пункты, а также объём, вес и тип груза для
грузоперевозок или количество пассажиров для пассажироперевозок. Должна
быть возможность погрузить и разгрузить заказ в/из конкретной машины. У
каждого транспортного средства должна быть возможность просмотреть
свободную грузоподъёмность, объём (либо количество мест) и заказы, которые
на данный момент загружены в машину.
Результат разработки:
- Объектно-ориентированное описание автопарка.
- Требуется промоделировать работу (создать несколько транспортных средств,
заправить и обслужить их, загрузить/разгрузить несколько заказов). В идеале –
с использованием автоматизированного цикла.
Оформление результата выполнения задачи:
- Листинг описания и примера с моделированием работы в одном TXT-файле.

Проект содержит 4 пакета:
package directories - пакет содержит справочники (несвязанные классы):
	FuelType (тип топлива) {DIESEL, LPG, GASOLINE}
	Cargo (тип груза) {FLUIDCARGO, FROZENPRODUCTS,  PRODUCTS,  LOOSECARGO, GOODS}

package order - 	пакет содержит классы, относящиеся к заказу. Любой заказ содержит в себе данные о отправителе, получателе, типе груза и своём состоянии.
	enum OrderState - состояние заказа (READYTOGO, INPROGRESS, COMPLETED}
	class Order являяесь суперклассом сожержит данные о пункте отправления заказа, пункте назначения заказа и его типе. Также у 
	класса присутствует счетчик, который считает кол-во созданных заказов и является уникальным идентификатором для любого заказа. Есть метод, позволяющий 
	распечатать информацию о заказе. У подклассов переопределен toString().
	subclasses CargoOrder и PassengersOrder расширяют Order и описывают заказ на перевозку груза и пассажиров, соответственно.
	Привязка заказа идёт к транспорту, и инициирует изменение заказа тоже транспорт. У каждого заказа может быть только один транспорт, который его берет.
	в тоже время в оном транспорте может быть несколько заказов (помещяются в список заказов). Загрузить заказ можно только в точке нахождения заказа при
	условии нахождения в этой же точке транспорта. Выгрузить можно только в точке назначения товара. Заказ передавать из транспорта в транспорт нельзя. 
	Для этого сначала требуется отменить заказ (перевести его в состояние READYTOGO), после чего его может взять другой транспорт. При приёме заказа 
	учитывается тип заказа, так, например, нельзя взять заказа на перевеоз пассажиров грузовику. Пример работы с заказами 	реализован для класса автобус 
	и грузопассажирский транспорт. Для остальных классов не реализовал, но суть будет такая же. 

package route - пакет содержит классы, относящиеся к маршруту
	class Point - описывает точку (назначения или отправления) и статический метод, который позволяет определить расстояние между точками 
	(в качестве примера расчёт расстояния по прямой между двумя точками).
	Может быть при необходимости дополнен методами, позволяющими оценочно узнать время преодоления расстояния между точками 
	в зависимости от транспортного средства, описать точку (наличие кондуктора, пункта пропуска и т.д.).
	class Itinerary - создержит связанный список точек назначения и отправления, фактически маршрут следования ТС. Вынесен в отдельный класс 
	и не привязан к транспортному средству, так как ТС может быть разное, а  маршрут может быть единым и повторяющимся. Полноценно не реализовал, но 
	суть вот в чем: создается, например, маршрут следования для автобуса. И автобус следует только этим маршрутом, отклоняться не может. Таким образом
	направиться за заказом в другой пункт, он не сможет, как и отклониться от маршрута в будущем. Также это можно учитывать при взятии заказов.

package transportsType - описывает транспортные средства:
	class Vehicle является суперклассом и содержит информацию относящцюся к любому транспортному средству:
	фирма-производитель, модель, дата производства, тип топлива (из справочника FuelType), расход топлива,  объем бака.
	Поле numberOfVehicleCount является счётчиком "созданных" объектов-транспорта.
	У объекта-транспорта есть поле numberOfVehicle, которое в момент создания становится равным счетчику созданных автомобилей, таким обраом достигается
	уникальность этого поля для кажого авто (аналог гос-номера или id в системе)
	Поле  HashMap<Integer, Order> ordersMap описывает заказы, "находящиеся" в данном ТС. Методы addOrder и removeOrder
	служат для добавления или удаления заказа в список заказов.
	Интерфейсы:
		interface Transport - описывает общие для любого транспорта методы, например ремонт и заправка топливом
		interface PassengerTransport (описывает пассажирский транспорт) и методы работы с ним (дезинфекция, чистка, определить кол-во свободных мест). 
		Содержит поле maxPassenger, по-умолчанию - 0, которое описывает максимально кол-во пассажиров.
		interface PassengerTransport extends Transport (описывает грузовой транспорт) и методы работы  с ним (пломбировка, загрузка, выгрузка, 
		определение свободного места). Содержит поле maxCargoWeight - максимальная грузоподъемность.
	Классы:
		class Bus extends Vehicle implements PassengerTransport описывает транспорт, который может перевозить только пассажиров. 
		Содержит поле maxPassenger - максимальное кол-во пассажиров, которое может взять транспорт
		class CargoPassenger extends Vehicle implements CargoTransport, PassengerTransport описывает транспорт, который может перевозить и пассажиров
		и грузы. 
		class Truck extends Vehicle implements CargoTransport - класс, описывающий некий средний грузовик. Его подклассами являются: 
			class TankTruck extends Truck - автоцистерна
			class TentTruck extends Truck - тентованный грузовик
			class RefrigeratorTruck extends Truck - грузовик-рефрижератор (можно указать поддерживаемую температуру)

Класс CarPark  - собственно автопарк, в котором работает вся логика: создаются машины, заказы, совершаются перевозки и т.д. 
Тесты реализованы в методже мэйн. Сначала ручное создание и проверка. Далее в Автоматическом режиме в цикле рандомно создаются два заказа, автобус 
с случайными параметрами, который пытается взять азказы, доставить их до места назначения и выгрузить.
Примечание:  большая часть методов результатом своей работы выдает строку, которую мы оцениваем, как действие. В реальной жизни, я бы скорее делал 
результатом булевое значение или, в зависимости от логики работы всей программы, возвращал номер транзакции или что-то вроде того, в случае неудачи - 
выкидывание исключения. Также было бы уменьшено кол-во параметров в конструкторе и методах. Исходим из цифры 3 парметра макимум.
